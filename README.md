# التوأم الرقمي لترعة الإسماعيلية — نسخة الباك إند

نموذج يؤدي نفس وظائف نموذج الإسماعيلية الأصلي، لكن **بمعمارية مختلفة جذريًا**:

| الأصلي | هذه النسخة |
|---|---|
| single-file PWA | باك إند + واجهة منفصلة |
| Firebase RTDB (NoSQL) | **Cloudflare D1 (SQL حقيقي)** |
| الحساب كله client-side | **مقسّم**: الثقيل على السيرفر، اللحظي على المتصفح |
| لا سجل تغييرات | **audit_log** كامل |
| GitHub Pages | **Cloudflare Workers** (سيرفر + API) |

نفس المعادلات والمعايرة الحقلية بالظبط: فم الترعة (خطي، خطأ 4.15%)، المنير وفم الترعة بمنحنيات قوة معايرة من القياسات الفعلية، ومانينج كـ fallback. **المنسوب خلف (wl_dn) يقود تصرف كل عقدة** — مش الأمام.

---

## النشر من Termux — خطوة بخطوة

```bash
# 1) المتطلبات (مرة واحدة)
pkg install nodejs
cd ismailia-backend
npm install

# 2) تسجيل الدخول لـ Cloudflare (هيفتح المتصفح)
npx wrangler login

# 3) إنشاء قاعدة البيانات
npm run db:create
#    انسخ الـ database_id اللي هيظهر وحُطّه في wrangler.toml

# 4) إنشاء الجداول + البيانات الأولية (على السحابة)
npm run db:init

# 5) النشر
npm run deploy
```

بعد النشر هيطلع لك رابط الشكل:
`https://ismailia-backend.<حسابك>.workers.dev`

افتحه — هتلاقي الواجهة شغّالة على نفس الرابط، والـ API تحت `/api/...`.

## نقاط الـ API

| المسار | الوظيفة |
|---|---|
| `GET /api/nodes` | كل العقد + معاملات المعايرة |
| `GET /api/readings?date=YYYY-MM-DD` | قراءات يوم + التصرف المحسوب لكل عقدة |
| `POST /api/readings` | حفظ/تحديث قراءة (upsert) |
| `GET /api/history?node_id=&days=30` | سلسلة زمنية لعقدة |
| `GET /api/profile?node_id=&date=` | منحنى backwater (مخزّن في الكاش) |
| `GET /api/balance?date=` | الميزان الكتلي (داخل/خارج/فجوة) |
| `GET /api/audit?limit=50` | سجل التغييرات |

## القطاعات

عرض قاع الترعة الرئيسية **يتدرّج تنازليًا** (40 م عند الفم → 10 م عند ك123)، كما هو فيزيائيًا صحيح — الترعة تضيق وهي رايحة. لكن **قطاع نقطة القياس** عند بعض الحجوز يتسع موضعيًا عند المنشأة، وهو القطاع الذي عُويرت عليه n من 236 قياس حقلي. القاعدة تحفظ الاثنين منفصلين:

- `bed_width` / `side_slope` = عرض الترعة الرئيسية المتدرّج (للعرض و backwater)
- `gauge_width` / `gauge_slope` = قطاع القياس المعاير (لـ fallback مانينج)

القيم المؤكدة من القياس:

| العقدة | عرض رئيسي | عرض قياس | منسوب القاع | n | المصدر |
|---|---|---|---|---|---|
| فم الترعة | 40 | 40 | 12.23 | 0.019 | 124 قراءة |
| المنير | 33.2 | 35 | **8.95** | 0.018 | 36 قراءة (القاع الفعلي وليس 11.50) |
| الصالحية | 21.7 | 56 | 6.72 | 0.038 | 76 قراءة، قطاع متسع عند المنشأة |

تصرف العقد الثلاث الأولى يأتي من **منحنيات المعايرة** (يُفضّل على مانينج). باقي الحجوز قطاعاتها تقديرية وتصرفها بمانينج من المنسوب خلف.

## إضافة عقد جديدة (مآخذ/فروع)

أضِف صفوفًا في `schema/seed.sql` ثم:
```bash
wrangler d1 execute ismailia-db --remote --command="INSERT INTO nodes (...) VALUES (...)"
```
